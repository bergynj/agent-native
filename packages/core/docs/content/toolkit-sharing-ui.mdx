# Toolkit Sharing UI

`@agent-native/toolkit/sharing` contains presentational sharing UI for badges,
share rows, role pickers, and base dialogs. It does not perform access checks or
mutations; core and app wrappers inject data and callbacks.

The most common low-level import is:

```tsx
import { VisibilityBadge } from "@agent-native/toolkit/sharing";
```

Core still owns the sharing runtime: share tables, resource registration, access
filters, visibility changes, invites, roles, and private access tokens for
agent-readable public surfaces. See the [Sharing Kit](/docs/toolkit-sharing) for
the full behaviorful contract.

Use core wrappers when a component needs to call sharing actions:

```tsx
import { ShareButton, ShareDialog } from "@agent-native/core/client";
```

Those imports remain compatible. Core wrappers keep doing the runtime work and
pass data and callbacks into Toolkit-owned UI.

## Boundary {#boundary}

Toolkit sharing UI can render badges, rows, base dialogs, and role pickers when
data and callbacks are injected. Core sharing code should own SQL, permissions,
request context, org membership lookups, and share-token minting.
